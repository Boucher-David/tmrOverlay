## Product Goal

Ship a Track Map overlay and local map-generation path together, plus pulled-forward Standings visibility and localhost browser-source routes for OBS. A normal user should not feel like the app silently mined their driving data with no explanation or visible benefit. If TMR derives a map from user telemetry, the user should see the result, know it stays local, and understand why the app generated it.

- Pit-lane line

## Confirmed Findings

IBT fields are sufficient for high-quality local map generation:

- `Lat`, `Lon`, and `LapDistPct` can generate recognizable track geometry.
- `LapDist` or session track length should remain the canonical distance source.
- `OnPitRoad` is present and should be used both to exclude pit coordinates from the racing line and to build a separate pit-lane geometry.

Laguna Seca high-volume probe:

- 594,501 records
- 125 complete filtered laps
- 563,441 selected racing-line points
- 1,200 generated bins
- 0 missing bins
- 1.04 m median per-bin spread
- 1.38 m p95 spread
- 3.03 m closure

Laguna one-lap vs five-lap probe, compared to all-lap median:

- One clean lap: 0.48 m median error, 2.23 m p95 error, 3.35 m max error.
- First five clean laps: 0.24 m median error, 1.21 m p95 error, 2.66 m max error.
- Weak one-lap file: 16 missing bins, 14.15 m p95 error, visibly poorer start/pit-area trace.
- Hard reject file: 11 records, 0.17 seconds, no valid moving points and no complete laps.

Pit-lane probe:

- 9,738 `OnPitRoad` samples
- 4,029 moving pit-road samples
- 3 drawable pit-lane passes
- 327.2 m median pit-lane path
- 0.58 m median spread
- 1.38 m p95 spread

## Data Model Sketch

Store generated maps in app-owned local storage, not capture folders:

```text
%LOCALAPPDATA%\TmrOverlay\track-maps\user
```

Map document should include:

- Schema/generation version
- Generated timestamp
- Track identity:
  - `TrackID`
  - `TrackName`
  - normalized config identity when present
  - fallback identity when `TrackConfigName` is empty
- `racingLine`:
  - local meter `x/y`
  - `LapDistPct`
  - optional cumulative distance scaled to `LapDist` or track length
- `pitLane`:
  - open local meter `x/y` path
  - open-lane progress or cumulative distance
  - start/end markers
- Quality:
  - confidence band
  - complete lap count
  - missing-bin count/pct
  - closure meters
  - length delta pct
  - repeatability median/p95 meters
  - pit-lane sample/pass counts and spread when available
- Provenance:
  - source `.ibt` path
  - source file size
  - source record count
  - capture id if known

Do not copy `.ibt` files into app storage by default.

## Confidence Draft

Reject:

- No complete positive-numbered lap
- No valid moving coordinates
- Large missing-bin coverage after generation

Low confidence:

- One complete lap
- No multi-lap repeatability metric
- Useful only as preview

Medium confidence:

- Two to four complete laps
- Low missing bins
- Low closure
- Stable length
- Limited repeatability evidence

High confidence:

- Five or more complete laps
- Zero or near-zero missing bins
- Closure under about 10 m
- Length delta under about 1%
- Multi-lap repeatability p95 under about 3 m

Treat complete generated maps as durable. Do not regenerate automatically unless no map exists, stored quality is below the runtime threshold, the map schema/generation version changes, or the user explicitly requests a rebuild. A later higher-confidence map can replace a weaker complete map.

## UX and Trust Rules

- Do not silently generate maps with no visible feature attached.
- Automatic generation is enabled by default behind the explicit `Build local maps from IBT telemetry` setting. Bundled app maps render when available; otherwise the disabled-generation path is the circle placeholder, and user-generated maps are ignored while the setting is disabled.
- Show plain-language local-data copy in settings/support.
- Store derived geometry only.
- Raw `.ibt` files remain external inputs and provenance, not app-owned data.
- Keep iRacing/Data API map assets as optional QA references only, not bundled runtime dependencies.
